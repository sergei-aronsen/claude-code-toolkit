🪄 Auto-format on edit opt-in

PostToolUse hook — runs prettier / eslint / pint / rubocop / ruff / gofmt / shfmt etc. after every Claude edit. Stack-aware via file extension. Silent on missing tools, never blocks Claude.

1) Verify the hook script is installed

The dispatcher lives at .claude/hooks/format-file.sh in every project (copied by /update-toolkit). Confirm it exists and is executable:

ls -la .claude/hooks/format-file.sh
# expected: -rwxr-xr-x ... format-file.sh

2) Wire it into .claude/settings.json

Add the second hook to the existing PostToolUse array (do NOT replace — the first hook handles activity logging):

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write|MultiEdit",
        "hooks": [
          {
            "type": "command",
            "command": "command -v jq >/dev/null 2>&1 || exit 0; f=$(jq -r '.tool_input.file_path // empty'); [ -n \"$f\" ] && echo \"[$(date +%H:%M:%S)] File modified: $f\" >> .claude/activity.log"
          },
          {
            "type": "command",
            "command": "command -v jq >/dev/null 2>&1 || exit 0; f=$(jq -r '.tool_input.file_path // empty'); [ -n \"$f\" ] && [ -x .claude/hooks/format-file.sh ] && .claude/hooks/format-file.sh \"$f\""
          }
        ]
      }
    ]
  }
}

3) Verify formatters are installed for your stack

The hook runs only the formatters present on PATH. Install the ones you need:

StackInstall
Next.js / Node.js / TypeScript npm i -D prettier eslint
Laravel / PHP composer require laravel/pint --dev
Rails / Ruby gem install rubocop
Python pip install ruff (or pip install black)
Go built-in gofmt; for imports: go install golang.org/x/tools/cmd/goimports@latest
Rust rustup component add rustfmt
Shell scripts brew install shfmt
SQL npm i -g sql-formatter

4) Disable globally or per-session

Globally:

export CLAUDE_FORMAT_HOOK=0

Fast mode (skip slow linters like eslint, run only formatters):

export CLAUDE_FORMAT_FAST=1

5) IDE conflict — important

If your editor has format-on-save enabled on the same file Claude is editing, you can hit a race where one tool's output overwrites the other. The hook ships with a per-file lock to deduplicate, but the safest setup is to disable IDE format-on-save while Claude is working:

  • VS Code: "editor.formatOnSave": false in .vscode/settings.json
  • JetBrains (PhpStorm/IntelliJ/WebStorm): Preferences → Tools → Actions on Save → uncheck "Reformat code"
  • Vim/Neovim: remove auto-format autocmds for the duration of the session

Alternative: keep IDE format-on-save and disable the Claude hook (CLAUDE_FORMAT_HOOK=0). Either tool alone is fine — both at once causes thrash.

6) Skipped paths

The hook auto-skips files inside node_modules/, vendor/, dist/, build/, .next/, coverage/, __pycache__/, .venv/, venv/ — Claude rarely edits there but if it does, formatting them produces noise.

7) Cost trade-off

For a typical 50-edit session:

  • Default mode: 25-100 seconds total formatter time (eslint dominates)
  • Fast mode: 5-10 seconds total (prettier-only, no linter pass)
  • Off: 0 seconds — but you must format manually before commit

If your pre-commit hook also runs prettier+eslint, the in-session formatting is redundant — turn off CLAUDE_FORMAT_HOOK and let pre-commit handle it.
