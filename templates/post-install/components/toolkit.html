🧰 Toolkit core no setup needed

Templates + slash commands + skills + agents installed into .claude/.

Daily commands

  • /update-toolkit — pull the latest toolkit content into .claude/, preserving local edits.
  • /update-deps — open the dependency dashboard (Layer 1/2/3 + MCP servers).
  • /audit — run one of 7 framework-aware audits.
  • /debug — 4-phase systematic debugger.

Commit .claude/ to git

The toolkit content is tracked per-project. Commit it so the same context follows the repo:

git add .claude/ CLAUDE.md
git commit -m "chore: install claude-code-toolkit"

Exception: .claude/settings.local.json is auto-gitignored — it carries per-developer credentials and must NOT be committed.
