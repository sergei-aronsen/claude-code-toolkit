🔌 Serena MCP

Symbol-aware code retrieval, refactoring and editing via LSP. Replaces text-search-and-replace with structured operations. oraios/serena — MIT, 23.9k stars.

What you need to configure

Nothing — Serena runs entirely locally with no API keys. The LSP servers download on first use per language.

⚠️ Disable the auto-opening dashboard tab

By default Serena spawns a web dashboard at http://127.0.0.1:24282/dashboard/ and opens it in your browser every time a Claude Code session starts. Combined with stale-process leaks (each session binds the next free port: 24282 → 24283 → 24284…), you end up with dozens of dashboard tabs after a day of work.

Fix once — edit ~/.serena/serena_config.yml:

# Disable both flags. Belt-and-suspenders: web_dashboard: false stops the
# server from starting at all; web_dashboard_open_on_launch: false ensures
# even if you re-enable the server later, it won't auto-open a tab.
web_dashboard: false
web_dashboard_open_on_launch: false

Or via sed:

sed -i.bak 's/^web_dashboard: true/web_dashboard: false/' ~/.serena/serena_config.yml
sed -i.bak 's/^web_dashboard_open_on_launch: true/web_dashboard_open_on_launch: false/' ~/.serena/serena_config.yml

If you ever want the dashboard for debugging, set web_dashboard: true but keep web_dashboard_open_on_launch: false — server runs, you open the tab manually only when needed.

Kill leaked Serena processes

Serena leaves zombies after Claude sessions. To clear them:

lsof -i -P 2>/dev/null | awk '/:2428[0-9]/ {print $2}' | sort -u | xargs -r kill

Worth running periodically (or in a SessionEnd hook).

Verify it's wired up

claude mcp list | grep serena

Then in a Claude Code session, ask "show me the Serena tools available" — you should see find_symbol, replace_symbol_body, find_referencing_symbols, list_dir, etc.

Per-project setup

The first time Serena touches a project, it creates .serena/ there with cached LSP indices. The directory grows, so add to .gitignore:

echo '.serena/' >> .gitignore

For per-project config (override defaults like language_backend, ignored_paths, read_only_memory_patterns), edit .serena/project.yml after first run.
