# Claude Guides — Шпаргалка

## Команды

| Команда | Что делает |
|---------|-----------|
| `/plan` | Создать план реализации перед кодом |
| `/debug` | Систематическая отладка (4 фазы) |
| `/verify` | Проверка перед коммитом: build, types, lint, tests |
| `/audit` | Аудит: security, performance, code, design, db |
| `/test` | Написать тесты для модуля |
| `/tdd` | Test-Driven Development: сначала тесты, потом код |
| `/fix` | Исправить конкретную проблему |
| `/refactor` | Улучшить структуру без изменения поведения |
| `/explain` | Объяснить как работает код или архитектура |
| `/doc` | Сгенерировать документацию |
| `/learn` | Сохранить урок в `.claude/rules/lessons-learned.md` (авто-загрузка) |
| `/context-prime` | Загрузить контекст проекта в начале сессии |
| `/checkpoint` | Сохранить прогресс в scratchpad |
| `/handoff` | Подготовить передачу задачи с описанием и следующими шагами |
| `/install` | Установить claude-guides в проект |
| `/worktree` | Управление git worktrees для параллельных веток |
| `/migrate` | Создать или отладить миграции БД |
| `/find-function` | Найти определение функции или класса |
| `/find-script` | Найти скрипты в package.json, Makefile и т.д. |
| `/docker` | Сгенерировать Dockerfile и docker-compose |
| `/api` | Спроектировать REST API, сгенерировать OpenAPI |
| `/e2e` | Сгенерировать E2E тесты с Playwright |
| `/perf` | Анализ производительности: N+1, бандл, память |
| `/deps` | Аудит зависимостей: безопасность, лицензии, устаревшие |

## Аудиты

Запуск: `/audit {тип}`:

| Тип | Что проверяет |
|-----|--------------|
| `security` | SQL-инъекции, XSS, CSRF, авторизация, секреты |
| `performance` | N+1 запросы, кеширование, lazy loading, размер бандла |
| `code` | Паттерны, читаемость, SOLID, DRY |
| `design` | UI/UX, доступность, адаптивность |
| `mysql` | Индексы, медленные запросы, performance_schema |
| `postgres` | pg_stat_statements, bloat, соединения |
| `deploy` | Чеклист перед деплоем |

## Рабочий процесс

### Три фазы (обязательно)

```text
RESEARCH (только чтение) --> PLAN (только scratchpad) --> EXECUTE (полный доступ)
```

### Уровни думания

| Уровень | Когда использовать |
|---------|-------------------|
| `think` | Простые задачи, быстрые фиксы |
| `think hard` | Многошаговые фичи, рефакторинг |
| `ultrathink` | Архитектурные решения, сложная отладка |

## Сценарии — Когда и что использовать

### Нашёл баг

```text
/debug описание бага
```

Claude сначала найдёт корневую причину, потом починит. После фикса: `/verify`

### Нужен code review

```text
/audit code
```

Для полного ревью: `/audit security`, затем `/audit performance`

### Хочу добавить новую фичу

```text
/plan описание фичи
```

Claude создаст план в scratchpad. После одобрения — выполнит. Потом: `/verify`

### Нужно написать тесты

```text
/tdd имя_модуля
```

Сначала пишет падающие тесты, потом минимальный код чтобы они прошли.

### Перед деплоем

```text
/verify
/audit security
/audit deploy
```

Все три — чтобы поймать проблемы до продакшена.

### Начинаю новую сессию

```text
/context-prime
```

Загружает контекст проекта, чтобы Claude сразу понимал кодовую базу.

### Передаю задачу другому разработчику

```text
/handoff
```

Создаёт описание: что сделано, текущее состояние, следующие шаги.

### Нужно безопасно отрефакторить

```text
/refactor целевой_код
```

Claude рефакторит сохраняя поведение. Всегда прогоняет тесты после.

### Хочу разобраться в чужом коде

```text
/explain path/to/file.ts
/explain поток авторизации
```

### Работа с базой данных

```text
/migrate создать таблицу users
/audit mysql
/audit postgres
```

### Проблемы с производительностью

```text
/perf
/audit performance
```

### Проверить зависимости

```text
/deps
```

Проверяет уязвимости, устаревшие пакеты, лицензионные проблемы.

### Нужен REST API

```text
/api спроектировать эндпоинты для users
```

Создаёт эндпоинты, OpenAPI спецификацию, CRUD-scaffold.

### Настроить Docker

```text
/docker
```

Генерирует Dockerfile и docker-compose для проекта.

### E2E тесты

```text
/e2e регистрация и логин пользователя
```

Генерирует тесты с Playwright для пользовательских сценариев.

## Полезные советы

- Всегда используй `/plan` перед большими фичами — экономит время и усилия
- Запускай `/verify` перед каждым коммитом — ловит проблемы рано
- Используй `/learn` после сложных решений — сохраняет знания для будущих сессий
- Начинай сессии с `/context-prime` — Claude работает лучше с контекстом
- Используй `/checkpoint` во время долгих задач — прогресс сохранится если сессия оборвётся
- `/debug` лучше чем «просто попробовать поправить» — систематический подход быстрее
