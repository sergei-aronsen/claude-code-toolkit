# Open Design Integration

Optional add-on: install the [nexu-io/open-design](https://github.com/nexu-io/open-design)
local-first prototyping environment via `scripts/setup-open-design.sh`. Open
Design is an open-source alternative to Anthropic's Claude Design that emits
HTML / PDF / PPTX / MP4 artifacts from prompts using its own agent runtime
plus 122 in-repo skills and 149 brand design systems.

## Why a separate installer (not a catalog entry)

Open Design is **not an MCP** and not a CLI tool. It is a standalone web app
that runs on `http://localhost:<port>` (default `7456`). It does not register
with `claude mcp` and does not expose tools to Claude Code over JSON-RPC.

The integration is therefore a thin wrapper around upstream's two supported
launch modes:

- **Docker** (default): `docker compose up -d` from `<repo>/deploy/`.
- **Source**: `corepack enable && pnpm install && pnpm tools-dev`.

`scripts/setup-open-design.sh` clones the upstream repo and dispatches.

## Quickstart

```bash
# Default: Docker mode, port 7456
bash scripts/setup-open-design.sh

# Custom port + directory
bash scripts/setup-open-design.sh --port 7777 --dir ~/projects/open-design

# Source mode (Node 24 + pnpm 10.33.x via corepack required)
bash scripts/setup-open-design.sh --mode source

# Preview, no side effects
bash scripts/setup-open-design.sh --dry-run

# Stop the stack (compose down — does not delete data)
bash scripts/setup-open-design.sh --stop
```

Idempotent — re-running clones once, then fast-forwards the existing
checkout. Re-up is near-instant (image cache hit).

## Prerequisites

| Mode | Required |
|---|---|
| docker (default) | Docker Desktop with `docker compose` v2 plugin |
| source | Node 24.x + corepack (selects pnpm `10.33.2` via `package.json#packageManager`) |

The script halts pre-flight if a prerequisite is missing — including under
`--dry-run`. Missing-tool errors are gating, not advisory.

## What gets installed

| Path | What | Owner |
|---|---|---|
| `$HOME/open-design/` | Upstream repo clone | nexu-io |
| `$HOME/open-design/deploy/.env` | Port override (only written if `--port` differs from default) | this script |
| Docker image `vanjayak/open-design:latest` | Runtime image | nexu-io |
| Docker volume `open-design-data` | Local app data | nexu-io |

Nothing is copied into `~/.claude/`. The toolkit's own settings, skills, MCP
registrations, and hooks are untouched.

## Skills + design systems

Open Design's `skills/` directory contains 122 SKILL.md files using the same
YAML frontmatter shape Claude Code skills use (`name`, `description`,
`triggers`). They also carry an extra `od:` block describing artifact mode,
platform, design-system requirements, and craft rules.

**These skills are NOT mirrored into `~/.claude/skills/`.** They are tightly
coupled to the Open Design runtime: each skill consumes an injected
`DESIGN.md` (one of 149 brand systems) plus universal `craft/` rules at
generation time. Without that runtime they produce broken HTML.

If you want Claude Code to drive Open Design, do it through the web UI on
`localhost:7456` — its agent runtime adapts to whatever local agent CLI is
installed (Claude Code, Codex, Cursor, Gemini CLI, etc.) and the skills
activate inside that flow.

## Security model

- **Network:** the upstream `deploy/docker-compose.yml` maps the host port
  to the container port; on a single-user laptop this binds to localhost
  through Docker Desktop's networking. On a multi-user host (shared dev
  box, cloud VM), verify with `docker compose ps` and adjust the compose
  port mapping if needed.
- **Data:** Docker named volume `open-design-data` lives under Docker's
  managed storage. `--stop` does NOT delete it; remove with
  `docker compose down -v` from `<repo>/deploy/` if you want a clean wipe.
- **Code execution:** Open Design renders HTML/PDF/PPTX/MP4 inside its own
  sandboxed preview pipeline. No code generated by it executes inside your
  Claude Code session.
- **Credentials:** Open Design supports BYOK (Bring Your Own Key) for
  agent calls when no local CLI is detected. If you supply an API key it
  is stored in the container's volume — keep that key separate from your
  Claude Code login.

## Stopping / removing

| Want | Command |
|---|---|
| Stop the stack | `bash scripts/setup-open-design.sh --stop` |
| Stop + delete app data | `cd ~/open-design/deploy && docker compose down -v` |
| Remove the clone | `rm -rf ~/open-design` |
| Remove the image | `docker image rm vanjayak/open-design:latest` |
