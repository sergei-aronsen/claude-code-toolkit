1. **NO `local` keyword at top-level (MAIN block).** `local` is legal ONLY inside function bodies. **Section C — REMOVE_LIST delete loop** (UN-01 hash-match invariant):

```bash
# UN-01: delete only files whose current sha256 matches recorded sha256.
# REMOVE_LIST was populated during the classification phase (18-01).
# PROTECTED_LIST is NEVER iterated here — base-plugin invariant.
#
# bash 3.2 + set -u safety: NO `local` (this is top-level), NO inline-default
# array guards — use array-length guard then unguarded expansion. abs_path is a
# plain variable (NOT `local abs_path`) because we are in MAIN block, not a
# function body.
DELETED_LIST=()
DELETE_FAILED_LIST=()

if [[ ${#REMOVE_LIST[@]} -gt 0 ]]; then
    log_info "Removing ${#REMOVE_LIST[@]} unmodified file(s)…"
    for rel in "${REMOVE_LIST[@]}"; do
        # Defense-in-depth: re-check protection at delete time
        if is_protected_path "$rel"; then
            log_warning "Refusing to delete protected path: $rel"
            continue
        fi
        # NOTE: NO `local` — this loop runs at MAIN block (top-level), not inside
        # a function. Using `local` here triggers shellcheck SC2168 + a bash
        # runtime error. Plain assignment is correct.
        abs_path="$rel"
        case "$rel" in
            /*) : ;;  # absolute already
            *)  abs_path="$CLAUDE_DIR/$rel" ;;
        esac
        if [[ -f "$abs_path" ]]; then
            if rm -f "$abs_path"; then
                DELETED_LIST+=("$rel")
            else
                DELETE_FAILED_LIST+=("$rel")
                log_warning "Failed to remove: $rel"
            fi
        fi
    done
fi
```
