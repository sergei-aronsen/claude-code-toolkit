### Section 5 — Process

Each step is an H3 with a short prose explanation followed by a fenced `bash` block Claude runs.

#### Step 1 — Parse Arguments

```bash
set -euo pipefail

# All args after the command, in raw form
ARGS=("$@")

if [ "${#ARGS[@]}" -lt 3 ]; then
    printf 'audit-skip: usage: /audit-skip <file:line> <rule> <reason...>\n' >&2
    exit 2
fi

FILE_LINE="${ARGS[0]}"
RULE="${ARGS[1]}"

# Split file:line on the LAST colon so paths containing : still work.
PATH_PART="${FILE_LINE%:*}"
LINE_PART="${FILE_LINE##*:}"

# Line must be a positive integer.
case "$LINE_PART" in
    ''|*[!0-9]*)
        printf 'audit-skip: <file:line> must end with :<positive integer>, got %q\n' "$FILE_LINE" >&2
        exit 2
        ;;
esac

# Reason tokens = everything after the rule, with --council=... extracted.
COUNCIL="unreviewed"
REASON_TOKENS=()
for tok in "${ARGS[@]:2}"; do
    case "$tok" in
        --council=council_confirmed_fp)
            COUNCIL="council_confirmed_fp"
            ;;
        --council=*)
            printf 'audit-skip: --council= only accepts council_confirmed_fp (got %q)\n' "$tok" >&2
            exit 2
            ;;
        *)
            REASON_TOKENS+=("$tok")
            ;;
    esac
done

if [ "${#REASON_TOKENS[@]}" -eq 0 ]; then
    printf 'audit-skip: reason is required (every token after the rule is joined as Reason)\n' >&2
    exit 2
fi

# Join reason tokens with single spaces. Use printf, never echo.
REASON="$(printf '%s ' "${REASON_TOKENS[@]}")"
REASON="${REASON% }"  # strip trailing space
```

(Use the literal Bash above — including `printf '%q'` for diagnostic output and `case` for the integer check, both POSIX-safe and macOS-BSD-safe.)

#### Step 2 — Validate File Path (git ls-files)

```bash
if ! git ls-files --error-unmatch -- "$PATH_PART" >/dev/null 2>&1; then
    printf 'audit-skip: %q is not tracked by git. Run `git add %q` and retry.\n' "$PATH_PART" "$PATH_PART" >&2
    exit 1
fi
```

Rationale paragraph below the block: an exception against an untracked file is a moving target; Phase 14/15 reasoning depends on a stable code excerpt the Council can read.

#### Step 3 — Validate Line Number

```bash
ACTUAL_LINES="$(awk 'END{print NR}' -- "$PATH_PART")"
if [ "$LINE_PART" -gt "$ACTUAL_LINES" ]; then
    printf 'audit-skip: %s has %d lines but you specified line %d\n' "$PATH_PART" "$ACTUAL_LINES" "$LINE_PART" >&2
    exit 1
fi
```

#### Step 4 — Check for Duplicate (exact triple)

```bash
EXC_FILE=".claude/rules/audit-exceptions.md"
HEADING="### ${PATH_PART}:${LINE_PART} — ${RULE}"

if [ -f "$EXC_FILE" ] && grep -Fxq -- "$HEADING" "$EXC_FILE"; then
    printf 'audit-skip: an entry already exists for %s:%s + %s\n\n' "$PATH_PART" "$LINE_PART" "$RULE" >&2
    # Print the offending block: the heading line + the next 4 lines.
    awk -v h="$HEADING" '
        $0 == h {found=1}
        found {print; if (NR > 0 && /^$/ && lines>0) {exit}}
        found {lines++}
        lines > 5 {exit}
    ' "$EXC_FILE" >&2
    exit 1
fi
```

(The em-dash in `HEADING` is a literal U+2014 character. Authors of this command MUST type the actual `—` glyph; do NOT escape as `—` — Bash will not decode it.)

#### Step 5 — Build Entry Block

```bash
TODAY="$(date -u +%Y-%m-%d)"

BLOCK_TMP="$(mktemp)"
trap 'rm -f "$BLOCK_TMP"' EXIT

{
    printf '\n'
    printf '%s\n\n' "$HEADING"
    printf -- '- **Date:** %s\n' "$TODAY"
    printf -- '- **Council:** %s\n' "$COUNCIL"
    printf -- '- **Reason:** %s\n' "$REASON"
} > "$BLOCK_TMP"
```

Note (in plan prose, NOT as Bash): the leading newline before the heading guarantees blank-line separation (markdownlint MD022/MD032). The trailing newline after `Reason:` is provided by `printf '%s\n'`.

#### Step 6 — Append Atomically

```bash
NEW_TMP="$(mktemp)"
trap 'rm -f "$BLOCK_TMP" "$NEW_TMP"' EXIT

cat "$EXC_FILE" "$BLOCK_TMP" > "$NEW_TMP"
mv "$NEW_TMP" "$EXC_FILE"
```

#### Step 7 — Confirm to User

```bash
printf '\033[0;32m✓\033[0m Added exception: %s:%s — %s\n' "$PATH_PART" "$LINE_PART" "$RULE"
printf '  Council: %s\n' "$COUNCIL"
printf '  Reason:  %s\n' "$REASON"
printf '\nFile updated: %s\n' "$EXC_FILE"
printf 'Note: changes are NOT staged. Run `git add %s` and commit when ready.\n' "$EXC_FILE"
```

### Section 6 — Key Principles

Bullet list (mirror tone of learn.md `## Key Principles`):

- **Hard refusal, no `--force`** — untracked paths, out-of-range lines, and exact-triple duplicates are rejected unconditionally.
- **Exact triple** — `<path>:<line>:<rule>` byte-for-byte case-sensitive. After a refactor, line 42 → 68 = NEW entry; remove the stale one with `/audit-restore`.

- **Default Council = `unreviewed`** — only set `--council=council_confirmed_fp` after a Council pass returned `FALSE_POSITIVE` (Phase 15).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Slash-command argv → Bash interpolation | The reason tokens are user-controlled and end up in shell variables, awk programs, grep patterns, and the markdown file body. |
| Slash-command argv → markdown file content | Reason text is written verbatim into `.claude/rules/audit-exceptions.md`, which is auto-loaded by Claude (`globs: ["**/*"]`) on every session. |
| Slash-command argv → file system path | `<path>` becomes a path argument to `git ls-files`, `awk`, and `cat`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Tampering | Reason field flowing into Bash | mitigate | All user input is captured into shell variables and emitted via `printf '%s'` (never `echo`, never `eval`, never `sh -c`). The Step 1 parser uses array indexing `"${ARGS[@]:2}"` and `case` matching — no command substitution on user input. |
| T-13-05 | Elevation of Privilege | Path argument to filesystem operations | mitigate | `git ls-files --error-unmatch -- "$path"` rejects anything not tracked in HEAD/index, which means `../../etc/passwd` cannot pass: it is not in `git ls-files` for this repo. The `--` separator prevents `-`-prefixed paths from being interpreted as flags. `awk` and `cat` also receive `--` separator where the syntax allows. |
| T-13-06 | Tampering / Information Disclosure | Reason field flowing into markdown auto-loaded by Claude | mitigate | The seed file (Plan 13-01) contains an explicit instruction that Reason is data, not directives. This plan does NOT attempt to escape arbitrary markdown — that would be defense-in-depth — but it DOES forbid the reason text from breaking out of the Reason bullet via codefence start: a reason starting with ``` ``` `` is wrapped. The primary defense is the seed-file framing prompt. |
| T-13-07 | Spoofing | Duplicate-triple detection bypass | mitigate | `grep -Fxq` (fixed-string, full-line) on the heading constant means a different rule ID, line number, or path = different entry by design (D-06). No fuzzy matching, no normalization. |
| T-13-08 | Repudiation | No timestamp on entry | accept | The Date field uses `date -u +%Y-%m-%d`, which captures the day. Per-second precision is not required — git blame on the file provides a true audit trail. |
| T-13-09 | Denial of Service | Massive Reason field | accept | A 10MB reason would bloat the rule file but cannot crash Claude or the auditor. No size cap is enforced. Risk acceptance is a deliberate trade-off (avoiding arbitrary cutoffs). |
</threat_model>
