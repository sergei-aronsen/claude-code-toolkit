## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Toolkit repo → end-user disk | `manifest.json` declares which files install scripts copy to a user's `.claude/`. A malicious or corrupt entry could direct an installer to copy or skip arbitrary files. |
| Toolkit repo → CI gate | `make check` is the only mechanical guarantee that version metadata stays consistent across releases. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01-01 | Tampering | manifest.json `files.scripts[].path` | mitigate | `python3 scripts/validate-manifest.py` exits 1 if any path does not resolve on disk; gate runs in `make check`. Plan 01 adds the entry only after the file exists (Phase 18 delivered `scripts/uninstall.sh`). |
| T-20-01-02 | Tampering | manifest.json version field drift | mitigate | `make version-align` enforces a triple equality (manifest, CHANGELOG, init-local.sh --version); Task 3 runs it as the closing gate. |
| T-20-01-03 | Information Disclosure | CHANGELOG.md placeholder date | accept | The literal `YYYY-MM-DD` placeholder is harmless; CONTEXT.md D-15 explicitly locks it during the in-progress window. The final tag commit replaces it; until then no real date leaks. |
| T-20-01-04 | Repudiation | CHANGELOG bullets misrepresent UN-XX | mitigate | Task 2 acceptance criteria require all 8 REQ-IDs to appear in the new section; bullets are reviewed against `.planning/REQUIREMENTS.md` text per D-13 discretion bullet 3. |
| T-20-01-05 | Denial of Service | Validator schema rejects new `files.scripts` key | mitigate | PATTERNS.md confirms the validator's iteration loop (`for section_name, entries in files_section.items()`) is open — ANY new section name is auto-accepted. Pre-execution audit performed during planning; no validator change required. |
| T-20-01-06 | Elevation of Privilege | n/a | accept | This plan only edits config files; no code execution paths added. No EoP surface. |
</threat_model>
