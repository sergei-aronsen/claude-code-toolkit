| Threat ID | Category | Component | Disposition | Mitigation Plan |

Documented in the `<interfaces>` block for Phase 3 to implement. |
| T-2-02 | Tampering (TOCTOU) | detect.sh filesystem probes | accept | Detection is a snapshot at source time; Phase 3/4 call sites may re-source before acting. No file is written by detect.sh — a race between probe and consumer install action is re-checked at install time by Phase 3. |

| T-2-S-03 | Denial of Service | detect.sh on a broken ~/.claude | accept | Filesystem probes use `2>/dev/null` + fallback empty strings; cannot hang. No network I/O. No subprocesses beyond `ls`, `jq`, `cat`. |

ASVS Level 1 applicability: V5 Input Validation partial (settings.json parsed via jq with safe defaults). No secrets, no auth, no crypto in this plan. `make check` + shellcheck enforce static analysis. No `high` severity threats: blocker threshold per security_requirement satisfied.
