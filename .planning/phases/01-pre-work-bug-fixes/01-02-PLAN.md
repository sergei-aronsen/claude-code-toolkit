<objective>
Add `< /dev/tty` guards to every interactive `read` call in `scripts/setup-council.sh` and add an early non-interactive guard so the script works correctly under `bash <(curl -sSL ...)` invocation and fails loudly (not silently) in CI.

Purpose: BUG-02 — without these guards, stdin for `read` is the curl-piped script body itself. The shell consumes script bytes as "user input," producing garbled answers or infinite hang. Every Supreme Council installer in the wild hits this.

Output: Three `read` calls patched, one early `/dev/tty` readable-check guard added near the top of the script. Pattern is copied verbatim from `scripts/init-claude.sh:84,430,479` per D-03.
</objective>

    **Site 1 — `scripts/setup-council.sh:93`** (GEMINI_CHOICE, has default "1"):

    Current:
    ```bash
    read -r -p "  Enter choice [1/2] (default: 1): " GEMINI_CHOICE

    ```

    Replace with:
    ```bash
    if ! read -r -p "  Enter choice [1/2] (default: 1): " GEMINI_CHOICE < /dev/tty 2>/dev/null; then

    fi
    GEMINI_CHOICE="${GEMINI_CHOICE:-1}"
    ```

    read -rs -p "  Enter Gemini API key (or press Enter to skip): " GEMINI_KEY < /dev/tty 2>/dev/null || true
    echo ""
