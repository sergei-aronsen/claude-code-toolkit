<objective>
Remove the silent `sudo apt-get update -qq && sudo apt-get install -y -qq tree 2>/dev/null` in `scripts/setup-council.sh:66`. Replace with: visible command line, user `[y/N]` confirmation, optional sudo invocation, and graceful degradation if declined or if install fails.

This is both a security transparency issue and a debugging black hole. `tree` is OPTIONAL for `brain.py` (D-11), so skipping is fine.

Output: One `elif` branch in Step 1 rewritten. No new dependencies.

    **Before** (current state, around lines 65-67 but verify current line numbers since 01-02 inserted the /dev/tty guard block above):

    ```bash
        elif command -v apt-get &>/dev/null; then
            sudo apt-get update -qq && sudo apt-get install -y -qq tree 2>/dev/null
            echo -e "  ${GREEN}✓${NC} tree installed via apt"
        else
    ```

    **After**:

    ```bash
        elif command -v apt-get &>/dev/null; then
            echo -e "  ${YELLOW}⚠${NC} tree not installed. To install, run:"
            echo -e "      sudo apt-get install tree"
            INSTALL_TREE=""
            if ! read -r -p "  Proceed with install? [y/N]: " INSTALL_TREE < /dev/tty 2>/dev/null; then
                INSTALL_TREE="N"
            fi
            if [[ "${INSTALL_TREE:-N}" =~ ^[Yy]$ ]]; then
                if sudo apt-get update && sudo apt-get install -y tree; then
                    echo -e "  ${GREEN}✓${NC} tree installed via apt"
                else
                    echo -e "  ${YELLOW}⚠${NC} apt install failed — brain.py structure analysis will be skipped"
                fi
            else
                echo -e "  ${YELLOW}⚠${NC} Skipped — brain.py structure analysis will be disabled"
            fi
        else
    ```

    Critical details:

    1. The command is ECHOED BEFORE the prompt so the user sees exactly what would execute. This is the core transparency fix (D-09).
    2. `read -r -p ... < /dev/tty 2>/dev/null` uses the same pattern as BUG-02 (for consistency with the rest of setup-council.sh after 01-02 landed). `/dev/tty` is REQUIRED here because this script may be run under `bash <(curl -sSL ...)`.
    3. `if ! read ...; then INSTALL_TREE="N"; fi` handles the edge case where `read` fails (EOF, /dev/tty missing) — default to NO.
    4. `"${INSTALL_TREE:-N}" =~ ^[Yy]$` — accept only `y` or `Y`; everything else (empty, n, anything) declines. This matches the `[y/N]` convention documented in the prompt.
    5. `sudo apt-get update && sudo apt-get install -y tree` — NO `-qq`, NO `2>/dev/null`. D-10: errors must be visible.
    6. The `-y` on apt-get install stays (auto-answer apt's own prompts once the user has consented at our prompt level). Removing `-y` would make apt hang on yet another prompt.
    7. BOTH failure paths (install failed / user declined) print a non-fatal warning and execution continues to the next step (D-11 — tree is optional).
    8. Use uppercase `INSTALL_TREE` (top-level script variable, not inside a function — no `local` declaration). This is consistent with `GEMINI_CHOICE`, `GEMINI_KEY`, etc. in the same script.

    No `exit` statement in this branch. If apt fails, we degrade to "no structure analysis" — brain.py must handle tree absence gracefully, which it already does per CONTEXT.md.
