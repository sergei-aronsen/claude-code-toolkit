<objective>
Remove the silent `sudo apt-get update -qq && sudo apt-get install -y -qq tree 2>/dev/null` in `scripts/setup-council.sh:66`. Replace with an advisory-only flow: print the exact command the user would need to run (`sudo apt-get install tree`), prompt `[y/N]` to acknowledge, then emit a non-fatal warning and continue. The script itself NEVER invokes `sudo`.

Purpose: BUG-04 — today the script silently escalates privileges (via `sudo` inside the script) and swallows any error output. Per D-09, the correct behaviour is for the script to stay out of the sudo business entirely: show the user what command would need to run, get acknowledgement, and degrade gracefully. This removes the privilege-escalation surface from the script and makes the install step fully transparent. `tree` is OPTIONAL for `brain.py` (D-11), so skipping is always safe.

Output: One `elif` branch in Step 1 rewritten. No new dependencies. No `sudo` invocations anywhere in the script.
</objective>
