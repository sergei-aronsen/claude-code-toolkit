<decisions>
## Implementation Decisions

### Architecture

- **D-01:** Sub-picker (`scripts/install.sh:1863`) keeps current contract: produces `TK_MCP_PRE_SELECTED` CSV of raw `MCP_NAMES`. Do NOT inject scope-glyph into `TUI_LABELS[]` (would corrupt CSV match per existing comment at 1853–1861).
- **D-02:** Scope-glyph rendering on sub-picker happens in `TUI_DESCS[]` suffix, not in `TUI_LABELS[]`. Format: ` [U]` / ` [P]` / ` [L]` after the existing `[MCP:✓ CLI:✗]` block. Read-only on sub-picker (informational only).
- **D-03:** New **scope-only main TUI screen** added: after sub-picker exports `TK_MCP_PRE_SELECTED`, a second `tui_checklist` runs with `TK_TUI_LOCK_SELECTION=1`. Selection (checkboxes) is locked; only Tab/s mutate `MCP_SELECTED_SCOPE[]`. Existing dead wiring at `install.sh:518–530` becomes the active wiring.
- **D-04:** Headless branch at `install.sh:453–471` (skipping main TUI when `TK_MCP_PRE_SELECTED` set) is **modified**, not removed: skip the LOCK-screen ONLY when caller passed `--mcp-scope` explicitly OR `--yes` is set. TTY-interactive run with sub-picker → always shows the LOCK-screen for scope choice.

### Scope detection (read-only display)

- **D-05:** Detect installed scope via three JSON sources, single-pass each, no per-row `claude mcp get` calls:
  - `~/.claude.json` → `.mcpServers[name]` exists → `user` scope
  - `<PROJECT_ROOT>/.mcp.json` → `.mcpServers[name]` exists → `project` scope
  - `~/.claude.json` → `.projects[<PROJECT_ROOT>].mcpServers[name]` exists → `local` scope
- **D-06:** Fallback when an MCP appears in multiple sources: `local` > `project` > `user` (most-specific wins). Reflects Claude Code's own resolution order.
- **D-07:** New helper `mcp_detect_installed_scope <name>` in `scripts/lib/mcp.sh`. Returns `user`|`project`|`local`|empty. Cached on first call (parses JSON once per shell).
- **D-08:** Installed-scope display: glyph appears in TUI_DESCS suffix on sub-picker AND TUI_LABELS on main lock-screen (where it's the primary signal). Format: `(installed: U)` / `(installed: P)` / `(installed: L)`.

### TUI lock mode

- **D-09:** New `TK_TUI_LOCK_SELECTION` env var. When set to `1`, `tui_checklist`:
  - Skips Space-toggle handling (Space becomes no-op, like Required rows)
  - Pre-fills `TUI_RESULTS[i]=1` for every row at entry
  - Footer hint replaces `Space toggle` with `selection locked`
  - Tab/Enter/Ctrl+C still active
- **D-10:** Lock-screen header: `Configure MCP scope (Tab toggles per-row, s sets all)`. Submit row: `[ Install with selected scopes ]`.

### Catalog default_scope patch

- **D-11:** Three changes in `scripts/lib/integrations-catalog.json`:
  - `comet-bridge`: `project` → `user` (single Comet profile per user, no blast radius)
  - `datadog`: `user` → `project` (prod monitoring, ack/mute incidents = real blast radius)
  - `posthog`: `user` → `project` (multi-product orgs: personal API key reads all projects)
- **D-12:** No automated migration of existing installs. Document in CHANGELOG: users who already have datadog/posthog at user-scope keep them; new installs default to project.

### Behavior for installed MCPs on lock-screen

- **D-13:** Already-installed MCPs (`TUI_INSTALLED[i]=1`) on the lock-screen render with detected scope from D-05/D-07 as the active glyph. Tab is **no-op** for these rows (mirrors Space immutability for installed rows). Justification: scope migration via `claude mcp remove + add` is destructive (re-OAuth, re-secret-prompt) and out of phase scope.
- **D-14:** A note line in the lock-screen footer when ≥1 installed row visible: `(installed rows show current scope, immutable — re-add to change)`.

### Out of scope (deferred)

- **DEF-01:** Migrating already-installed MCPs to a different scope. Requires `remove --scope X && add --scope Y`. Punt to a follow-up phase.
- **DEF-02:** Per-project scope storage redesign (already shipped in v6.4 via global slot). No further work here.
- **DEF-03:** Skills/CLI sub-picker scope picker. Out of phase scope (skills don't have scope concept; CLI is always global).

### Claude's Discretion

- Test fixture style: Bash `set -e` + `assert_eq` matches existing `scripts/tests/` pattern. Use it.
- Exact wording of footer hints / banner: pick clearest 1-line copy that fits in 100-col terminal.
- Color choice for installed-scope glyph (suggest dim-cyan to distinguish from selected-scope green).

</decisions>
