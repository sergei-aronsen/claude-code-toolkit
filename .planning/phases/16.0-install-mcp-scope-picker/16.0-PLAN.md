# Phase 16.0: Install MCP Scope Picker — Plan

**Phase:** 16.0-install-mcp-scope-picker
**Target tag:** v6.16.0
**Planned:** 2026-05-10
**Depends on:** v6.15.3 (HEAD = de7f858)
**Estimated LOC:** ~800 (350 src + 350 tests + 100 docs; +200 for Back-removal)
**Estimated tasks:** 12 (added T-08a/b/c for Back-removal scope expansion)

---

## Goals

1. Surface the per-row MCP scope picker that has been dead-code since Phase 39 (`scripts/install.sh:518–530` wires it but headless branch at 453–471 swallows it).
2. Add a dedicated **lock-screen** between sub-picker and dispatch: selection is locked (came from sub-picker), only Tab/s mutate `MCP_SELECTED_SCOPE[]`.
3. Show installed-MCP scope (read-only) on both sub-picker (in `TUI_DESCS` suffix) and lock-screen (in label).
4. Patch three catalog defaults: `comet-bridge` → `user`, `datadog` → `project`, `posthog` → `project`.
5. **Remove Back navigation from all TUI screens** (reverses Phase 36-A 2026-05-02 feature per user request 2026-05-10). Linearize sub-picker state machine.
6. Bump to v6.16.0, ship via PR, tag, GitHub Release, CHANGELOG.

## Non-Goals (Deferred)

- DEF-01: in-place re-scope of installed MCPs (would require `claude mcp remove + add` with re-OAuth/secret prompt UX).
- DEF-02: per-MCP per-project scope persistence (already partially solved by v6.4 global-slot).
- DEF-03: skill / CLI sub-picker scope picker.
- DEF-04: replace JSON parsing with `claude mcp get` per row (slow on 28 entries).

### T-02 — `mcp_detect_installed_scope` helper

**Files:**
- `scripts/lib/mcp.sh` (new function ~50 LOC)
- `scripts/tests/test-mcp-detect-installed-scope.sh` (new)

**Approach:**

Add new function at end of `scripts/lib/mcp.sh`:

```bash
# mcp_detect_installed_scope <name> — print "user"|"project"|"local"|"" for the
# given MCP name based on three JSON sources (parsed once per shell, cached):
#
#   ~/.claude.json        → .mcpServers[name]                 → user scope
#   $TK_MCP_PROJECT_ROOT/.mcp.json → .mcpServers[name]        → project scope
#   ~/.claude.json        → .projects[$PROJECT_ROOT].mcpServers[name] → local scope
#
# Resolution order (most-specific wins): local > project > user. Reflects
# Claude Code's own scope precedence so display matches `claude mcp get`.
#
# Test seams:
#   TK_MCP_DETECT_USER_JSON   — override path to ~/.claude.json
#   TK_MCP_DETECT_PROJECT_JSON — override path to <project>/.mcp.json
#   TK_MCP_DETECT_PROJECT_ROOT — override $PWD for projects[<root>] lookup
mcp_detect_installed_scope() { ... }
```

Cache strategy: parse JSON once per shell into 3 newline-separated lists (`_MCP_SCOPE_USER_LIST`, `_MCP_SCOPE_PROJECT_LIST`, `_MCP_SCOPE_LOCAL_LIST`). Lookup is grep against the lists. Bash 3.2 compatible (no associative arrays).

**Tests:** `test-mcp-detect-installed-scope.sh`:
- Fixture `~/.claude.json` with `mcpServers.context7` → expect `user`
- Fixture `.mcp.json` with `mcpServers.stripe` → expect `project`
- Fixture `~/.claude.json` with `projects[/foo].mcpServers.dbhub` → expect `local`
- Same MCP in user + local → expect `local` (precedence)
- Missing JSON files → expect empty
- Unknown name → expect empty
- Malformed JSON → expect empty (silent fallback, no crash with `set -u`)

**Verify:** 7 assertions pass.

**Commit:** `feat(mcp): add mcp_detect_installed_scope helper`

### T-05 — Lock-screen wiring after sub-picker

**Files:**
- `scripts/install.sh` (~80 LOC new code + 10 LOC modifying headless branch)

**Approach:**

After the sub-picker exports `TK_MCP_PRE_SELECTED` (around line 1874), and BEFORE the headless branch at line 453 fires:

1. New branch in the main MCP TUI dispatcher: when `TK_MCP_PRE_SELECTED` is set AND TTY is interactive AND `--mcp-scope` was NOT passed AND `--yes` is NOT set → render the lock-screen.

```bash
# In the existing TUI block in install.sh (around line 488)
elif [[ -r "$_install_tty_src" ]] \
   && [[ -n "${TK_MCP_PRE_SELECTED:-}" ]] \
   && [[ -z "${TK_MCP_SCOPE_CLI:-}" ]] \
   && [[ "$YES" -ne 1 ]]; then
    _run_mcp_scope_lock_screen   # new function below
fi
```

2. New function `_run_mcp_scope_lock_screen`:

```bash
_run_mcp_scope_lock_screen() {
    # Rebuild TUI_LABELS / MCP_SELECTED_SCOPE filtered to the CSV from sub-picker
    # Reuse mcp_status_array, then trim arrays to PRE_SELECTED entries
    mcp_status_array
    _mcp_filter_arrays_by_pre_selected   # new helper

    # Re-label rows to include scope glyph (from catalog default OR detected installed)
    local _i
    for ((_i=0; _i<${#TUI_LABELS[@]}; _i++)); do
        local _name="${MCP_NAMES[${TUI_TO_MCP_IDX[$_i]}]}"
        local _detected
        _detected=$(mcp_detect_installed_scope "$_name")
        if [[ -n "$_detected" ]]; then
            # Installed — lock the scope choice for this row (T-06 below)
            MCP_SELECTED_SCOPE[$_i]="$_detected"
            TUI_INSTALLED[$_i]=1   # mark immutable for both Space (locked anyway) and Tab
        fi
    done

    # Wire Tab/s + lock selection
    TK_TUI_LOCK_SELECTION=1
    TUI_HEADER_KEY="s"
    TUI_HEADER_FN="mcp_toggle_scope"
    TUI_ROW_KEY=$'\t'
    TUI_ROW_FN="mcp_cycle_row_scope_locked"   # new wrapper that no-ops on installed rows
    TUI_HEADER_TEXT="Configure MCP scope (Tab toggles per-row, s sets all)"

    if ! tui_checklist; then
        # Cancel — exit 0 (consistent with sub-picker cancel)
        echo "Install cancelled at scope picker."
        exit 0
    fi
    unset TK_TUI_LOCK_SELECTION TUI_HEADER_KEY TUI_HEADER_FN TUI_ROW_KEY TUI_ROW_FN TUI_HEADER_TEXT

    # MCP_SELECTED_SCOPE is now set per-row. The downstream dispatcher
    # (around line 600) reads it as today.
}
```

3. **Headless branch (lines 453–471)** stays for `--mcp-scope` + `--yes` paths. Add a comment explaining the bypass.

**Tests:**

- `--mcp-scope project` + sub-picker → headless branch wins, lock-screen NOT shown.
- `--yes` + sub-picker → headless branch wins, lock-screen NOT shown.
