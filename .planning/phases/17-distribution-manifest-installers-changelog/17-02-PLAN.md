<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo → user filesystem | Both installers write to `~/.claude/council/prompts/`. The downloaded prompt is consumed verbatim by `brain.py` and sent to remote Gemini + ChatGPT. |
| user filesystem → remote LLMs | A stale or corrupted prompt would propagate to Gemini/ChatGPT and produce wrong audit-review verdicts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-02 | Tampering / Spoofing | `~/.claude/council/prompts/audit-review.md` (consumed by `brain.py --mode audit-review`) | mitigate | mtime-aware refresh: re-running `setup-council.sh` after a toolkit upgrade overwrites the prompt only when the shipped version's mtime is newer. A user who deliberately edits the local prompt and then upgrades the toolkit may have it overwritten when the upstream mtime is newer — that's the intended behavior. A `--force` opt-out flag is deferred to a future hardening pass. Curl uses `-sSLf` (already enforced project-wide) so HTTP 4xx/5xx fails the download cleanly without a partial-write risk. |
</threat_model>
