<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test sandbox → host filesystem | Test creates `$SANDBOX` under `/tmp/`. EXIT trap removes it. |
| TK_UNINSTALL_HOME variable | Set by test to `$SANDBOX`; uninstall.sh trusts it for path resolution. Test-only seam. |
| heredoc fixture content → CLAUDE.md | Test writes user-controlled content (from the test source itself, not external input) to a sandbox file. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-03-01 | Tampering | `trap 'rm -rf "$SANDBOX"' EXIT` — `$SANDBOX` empty → host wipe | mitigate | `mktemp -d` returns non-empty path on success; `set -euo pipefail` aborts before trap if mktemp fails. Use `trap 'rm -rf "${SANDBOX:?}"' EXIT` with parameter expansion `:?` to fail fast if the variable somehow becomes empty before the trap fires. |
| T-19-03-02 | Spoofing | A malicious `TK_UNINSTALL_LIB_DIR` could load attacker-controlled lib | accept | Test sets `TK_UNINSTALL_LIB_DIR="$REPO_ROOT/scripts/lib"` from a trusted, repo-rooted absolute path. Not a user-facing attack surface. |
| T-19-03-03 | Information disclosure | Output excerpt on assertion failure could expose sandbox paths | accept | Sandbox paths under `/tmp/uninstall-state.XXXXXX` carry no secrets. Test-only diagnostic surface. |
| T-19-03-04 | Denial of service | A sentinel fixture with a billion lines could blow up `awk` strip | accept | Test fixture is bounded (a few hundred bytes). Not a real DoS surface. The strip-helper's awk implementation is O(1)-memory streaming regardless of input size. |

**Severity:** No HIGH findings. One `mitigate` (sandbox cleanup safety via `${SANDBOX:?}` expansion); others `accept` with rationale.
</threat_model>
