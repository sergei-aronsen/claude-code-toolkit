# Plan 34-01: Category-grouped rendering + per-component status

## Goal

Render the 20-entry catalog grouped by `category` field with per-component status columns (`MCP: ✓/✗/⊘ already` + `CLI: ✓/✗/⊘ already / —`).

## Schema reminder

```jsonc
{
  "schema_version": 2,
  "categories": ["docs-research", "backend", ...],
  "components": {
    "mcp": { "<name>": { "name", "display_name", "category", "install_args", "env_var_keys", "requires_oauth", "description", "unofficial?: bool" } },
    "cli": { "<name>": { "detect_cmd", "install": {"darwin", "linux"}, "post_install_hint" } }
  }
}
```

`components.cli.<name>` is present only when entry has CLI. Read with `// empty` jq fallback.

## Tasks

### Task 1: Add category grouping to mcp.sh catalog loader

In `scripts/lib/mcp.sh`, extend `mcp_catalog_load` (or add `mcp_catalog_load_grouped`) to populate parallel arrays:

- `MCP_CATEGORY[]` — entry's category (from MCP block)
- `MCP_HAS_CLI[]` — `1` if `components.cli.<name>` exists, else `0`
- `MCP_UNOFFICIAL[]` — `1` if `components.mcp.<name>.unofficial == true`, else `0`

Read order: by category (canonical 10-list order), within category alphabetical by entry name. The canonical 10-list is loaded from `.categories` array in catalog.

```bash
# Pseudocode in mcp.sh
mcp_categories_load() {
  local catalog_path="$1"
  CATEGORIES_ORDER=()
  while IFS= read -r cat; do
    CATEGORIES_ORDER+=("$cat")
  done < <(jq -r '.categories[]' "$catalog_path")
}
```

### Task 2: Add per-component status detection

New function `mcp_status_detect` (extends or replaces `mcp_status_array`):

```bash
mcp_status_detect() {
  # Populates parallel arrays:
  # MCP_STATUS[]   — "installed"|"absent"|"unknown"
  # CLI_STATUS[]   — "installed"|"absent"|"na"  (na when entry has no CLI block)
  
  # MCP detect: parse `claude mcp list 2>/dev/null` once, build set of installed MCP names
  local installed_mcps=""
  if command -v claude >/dev/null 2>&1; then
    installed_mcps=$(claude mcp list 2>/dev/null || echo "")
  fi
  
  # Per-entry detect
  local i name detect
  for ((i=0; i<${#MCP_NAMES[@]}; i++)); do
    name="${MCP_NAMES[$i]}"
    if [[ -z "$installed_mcps" ]]; then
      MCP_STATUS+=("unknown")
    elif printf '%s\n' "$installed_mcps" | grep -q "^${name}\b"; then
      MCP_STATUS+=("installed")
    else
      MCP_STATUS+=("absent")
    fi
    
    # CLI detect — only if entry has CLI block
    if [[ "${MCP_HAS_CLI[$i]}" == "1" ]]; then
      detect=$(jq -r --arg n "$name" '.components.cli[$n].detect_cmd' "$CATALOG_PATH")
      if command -v "$detect" >/dev/null 2>&1; then
        CLI_STATUS+=("installed")
      else
        CLI_STATUS+=("absent")
      fi
    else
      CLI_STATUS+=("na")
    fi
  done
}
```

Bash 3.2: parallel arrays, no associative arrays. `MCP_NAMES[]` already populated by `mcp_catalog_load`.

### Task 3: Render TUI with category groups + status columns

Modify `mcp_status_array` (renderer) or add `integrations_render` to:

1. Iterate `CATEGORIES_ORDER[]`.
2. For each category with ≥1 entry, print header: `── <Category Display> ──` (e.g., `── Backend ──`).
3. Print entries in this category alphabetical, each row format:
   ```
   [ ] supabase                Postgres backend...        MCP:✓  CLI:✗
   ```
   - Unofficial entries get yellow `!` glyph: `[ ]! notebooklm`
   - Status glyphs: `✓ installed`, `✗ absent`, `⊘ unknown`, `—` for CLI:na

Use `tui.sh` primitives if they exist. Adapt status column width to terminal width (`tput cols` if available, fallback 80).

Category display names: title-case the canonical key (e.g., `docs-research` → `Docs Research`, `project-management` → `Project Management`).
