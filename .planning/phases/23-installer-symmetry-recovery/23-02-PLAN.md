KEEP-01 — add `--keep-state` (and `TK_UNINSTALL_KEEP_STATE=1` env var) to `scripts/uninstall.sh` so users who answered N to every modified-file prompt can re-run the uninstaller and see the remaining files instead of a silent no-op. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Shell environment → uninstall script argparse | Caller exports `TK_UNINSTALL_KEEP_STATE=1` env var or passes `--keep-state` flag; the value is read into a script-local boolean integer with NO direct interpolation |
| Argparse → state-delete gate | `[[ $KEEP_STATE -eq 0 ]]` integer-comparison only; no command substitution, no eval |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-02-01 | Tampering (input) | `--keep-state` argparse clause in `scripts/uninstall.sh` | accept | The clause is value-less (boolean flag); user input goes only into `case "$arg" in --keep-state)` literal-match. No interpolation, no eval. Threat surface: ZERO. |
| T-23-02-02 | Tampering (env) | `TK_UNINSTALL_KEEP_STATE` env var via `KEEP_STATE=${TK_UNINSTALL_KEEP_STATE:-0}` | accept | Env-var value is consumed only by `[[ $KEEP_STATE -eq 0 ]]` integer test.
