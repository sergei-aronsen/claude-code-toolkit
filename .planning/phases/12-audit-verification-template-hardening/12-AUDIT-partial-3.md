## Claims 11-15

| Claim # | Claim Summary | Status | Evidence | Action |
|---------|---------------|--------|----------|--------|
| AUDIT-11 | No template integrity checksum — no `manifest.hash` | FALSE | `scripts/update-claude.sh:102-104` computes `MANIFEST_HASH=$(sha256_file "$MANIFEST_TMP")` on every update run. `scripts/update-claude.sh:167-182` computes per-file sha256 and compares against stored hashes in `toolkit-install.json:installed_files[].sha256`. `scripts/lib/state.sh:43+` writes sha256 per installed file. `scripts/update-claude.sh:757-761` writes `manifest_hash` field to `toolkit-install.json` after each run. Full integrity pipeline exists: per-file sha256 stored at install time, compared on every update. No separate `manifest.hash` file, but the functional equivalent is implemented inside `toolkit-install.json`. | → No action |
| AUDIT-12 | Markdown commands as templates without linting — required sections/frontmatter/step markers not enforced | PARTIAL | `Makefile:105-138` `validate` target greps `templates/*/prompts/*.md` audit files for `QUICK CHECK` and `САМОПРОВЕРКА\|SELF-CHECK` markers. `.github/workflows/quality.yml:37-68` `validate-templates` job enforces same markers in CI. `ls commands/` shows 30 command files. `grep -q "^## Purpose"` check on `commands/audit.md`, `commands/plan.md`, `commands/api.md` — all 3 have `## Purpose` and `## Usage` headings, but these are not enforced by any Makefile target or CI job. Gap: audit-prompt template validation exists for `templates/*/prompts/*.md` but `commands/*.md` have no equivalent structural enforcement. No `validate-commands` target in `Makefile:1` `.PHONY` list. | → HARDEN-A-01 (proposed) |
| AUDIT-13 | No dependency graph between templates — rag requires memory, installer doesn't enforce | FALSE | `grep -n "requires\|depends\|dependency" manifest.json` → not found as top-level keys. `python3 -c "import json; m=json.load(open('manifest.json')); print(list(m.keys()))"` → `['manifest_version', 'version', 'updated', 'description', 'sp_equivalent_note', 'files', 'inventory', 'claude_md_sections', 'templates']`. No `rag` template exists (`ls templates/` → base, global, go, laravel, nextjs, nodejs, python, rails). The specific dependency example (rag-requires-memory) is inapplicable — no rag template. Framework templates are self-contained and mutually exclusive (one per install); no inter-template dependencies exist by design. | → No action |
| AUDIT-14 | No uninstall semantics — can't remove template safely | REAL | `ls scripts/ | grep -i uninstall` → no uninstall script found. `grep -rn "uninstall\|remove\|--uninstall" scripts/` → no `--uninstall` flag in any installer. `scripts/migrate-to-complement.sh` migrates between modes (v3.x → complement) with per-file [y/N] prompt but is NOT an uninstall — it removes duplicates against SP/GSD, not the toolkit itself. No `uninstall-claude.sh` or `init-claude.sh --uninstall` path exists. User cannot remove the toolkit without manual file deletion. | → HARDEN-C-04 (deferred v4.2+) |
| AUDIT-15 | No template provenance metadata — no `installed_templates.json` post-install | PARTIAL | `find . -name "installed_templates.json"` → not found. `scripts/init-claude.sh:450` calls `write_state "$MODE" "$HAS_SP" ... "$INSTALLED_CSV" "$SKIPPED_CSV"` which writes `~/.claude/toolkit-install.json` with `version`, `mode`, `detected`, `installed_files` (with per-file sha256), `skipped_files`, `installed_at` fields (from `scripts/lib/state.sh:43+` write_state function). Gap: `toolkit-install.json` records installed files and mode but does NOT record which framework template was used (`FRAMEWORK` variable is not passed to `write_state`). A future auditor cannot determine from `toolkit-install.json` alone whether `laravel` or `nodejs` template was installed. | → HARDEN-C-05 (deferred v4.2+) |
