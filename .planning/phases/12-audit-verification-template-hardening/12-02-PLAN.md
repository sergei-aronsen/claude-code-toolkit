<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| New Makefile recipes | Shell commands executed by `make check` from the user's repo checkout. Input: repo files only (no user-supplied paths). Trust domain: same as existing Makefile targets. |
| New Python validators | `scripts/validate-*.py` may walk the filesystem under REPO_ROOT. Input: repo files only. Output: stderr messages. No network calls, no subprocess spawning beyond existing patterns. |
| CI workflow | `.github/workflows/quality.yml` runs on `ubuntu-latest` GitHub runners with `permissions: contents: read`. New steps inherit the hardened permission boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05 | Tampering | new Makefile `validate-*` targets | mitigate | Each target uses the ERRORS accumulator pattern from `agent-collision-static` — fixed glob patterns (`commands/*.md`, `manifest.json`), no user-controlled interpolation. No `eval`, no `$(<var>)` command substitution on user input. Grep patterns are literal strings defined in the target body. |
| T-12-06 | Injection | grep arguments in new Makefile recipes | mitigate | File paths come from literal glob expansion over known repo directories (`commands/*.md`, `templates/*/CLAUDE.md`). No environment-variable interpolation into grep patterns. No user input reaches the recipe. Pattern matches existing `validate-base-plugins` target (Makefile:143-149) which is already accepted. |
| T-12-07 | Injection | new scripts/validate-*.py file walk | mitigate | `REPO_ROOT` resolved via `os.path.abspath(__file__)` (not user-supplied); file walks use `os.listdir` + explicit `.md` suffix filter (no `..` traversal, no symlink following). Mirrors `scripts/validate-manifest.py` which is already audited. |
| T-12-08 | Denial of Service | CI runtime | accept | New CI steps perform grep/glob over a fixed-size repo (≤1000 markdown files); worst-case runtime is seconds. Matches existing `make check` cost. |
| T-12-09 | Information Disclosure | validator stderr messages | accept | Error messages print file paths relative to REPO_ROOT only — no secrets handled, no env-var dumps. Same error-message surface as existing `validate-manifest.py`. |
| T-12-10 | Elevation of Privilege | CI permissions | mitigate | New CI steps inherit the workflow-level `permissions: contents: read` (quality.yml line 9). No new permission grants. SHA-pinned `actions/checkout` reused from parent job context; no new third-party actions introduced. |
| T-12-11 | Tampering | REQUIREMENTS.md + 12-AUDIT.md edits | mitigate | All edits land in a single git commit with conventional-commit message and the paper trail from Plan 12.1 gate decisions. `markdownlint` enforces file structure; grep-based acceptance criteria enforce row counts and status strings. |
</threat_model>
