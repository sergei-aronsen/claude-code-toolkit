10. `unofficial: true` set is `{notebooklm, telegram}` exactly
11. NO `sequential-thinking` entry
12. NO `sudo` token in any `install.linux`/`install.darwin` string

### Task 2: TEST-02 — test-cli-installer.sh

Assertions (≥8):
1. `cli_detect bash` → 0 (real builtin)
2. `cli_detect __nonexistent_xxx` → 1
3. `cli_install foo "echo darwin-cmd" "echo linux-cmd"` with `TK_CLI_UNAME=Darwin` runs darwin cmd
4. Same with `TK_CLI_UNAME=Linux` runs linux cmd
5. `TK_CLI_UNAME=FreeBSD` → exit 2 with stderr message

8. `cli_post_install_hint "test hint"` writes `→ Next: test hint` to stderr only (stdout empty)
9. No-sudo invariant: grep `sudo` in cli-installer.sh source — must not appear (or appear only in comments)

### Task 3: TEST-03 — test-integrations-tui.sh

Create `scripts/tests/test-integrations-tui.sh`. ≥15 NEW assertions.

Coverage areas (per CONTEXT.md / Phase 34 outputs):
1. Category headers render in canonical order
2. Categories with 0 entries skipped
3. Per-row MCP status: when `claude mcp list` is mocked to include "supabase", supabase row shows `⊘` / installed glyph
4. Per-row CLI status: when `command -v wrangler` mocked to succeed, cloudflare row shows `⊘`
5. Unofficial entries (notebooklm, telegram) render with `!` glyph
6. Unofficial confirm prompt fires when entry selected — `TK_INTEGRATIONS_TTY_SRC` test seam
7. Unofficial declined → entry skipped from install
8. `--mcp-only` and `--cli-only` mutex → exit 2 with stderr "mutually exclusive"
9. `--mcp-only` skips CLI dispatch step
10. `--cli-only` skips MCP dispatch step
11. `--yes` bypasses unofficial prompt
12. Summary table prints at end with Entry/MCP/CLI/Notes columns
13. Summary counters: Installed/Skipped/Failed numbers match outcomes
14. Browse-mode (claude absent) preservation: `--mcps --yes` exits 0, no CLI installs triggered
15. `--mcps` deprecation note prints to stderr, install proceeds normally

Use `TK_TUI_FLAT=1` if such a seam was added by Phase 34, OR just run the TUI in `--yes` non-interactive mode where possible.

Mock claude CLI by overriding PATH:
