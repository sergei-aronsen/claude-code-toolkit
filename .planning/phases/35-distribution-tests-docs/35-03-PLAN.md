### Task 4: Global vs per-project boundary (DOCS-02)

```markdown
## Global vs per-project

This catalog installs **dev-machine globals** â€” stuff Claude needs to call mid-conversation:
- MCP servers register globally with Claude (`claude mcp add`)
- CLI tools land in `/usr/local/bin` (brew) or `~/.local/bin` (user-space)

Toolkit does NOT install per-project SDKs. Things like:
- `@supabase/supabase-js` â€” your project's `package.json`
- `stripe-node` â€” your project's `package.json`
- `@aws-sdk/client-s3` â€” your project's `package.json`

These are **dependencies of the application you're building**, not of Claude Code. Add them with `npm install` / `composer require` / `pip install` per project, not via this toolkit.

Rule of thumb:
| Layer | Where | Example |
|-------|-------|---------|
| MCP server | global (`~/.claude/mcp/`) | `claude mcp add supabase ...` |
| CLI dev tool | global (`/usr/local/bin`, `~/.local/bin`) | `wrangler`, `supabase`, `stripe` |
| SDK | per-project (`node_modules/`) | `@supabase/supabase-js` |
| Project config | per-project (`.env`, app config) | `SUPABASE_URL`, `SUPABASE_KEY` |

If you find yourself wanting to install an SDK via this toolkit, the answer is no â€” that's your project's concern.
```

### Task 6: Troubleshooting

```markdown
## Troubleshooting

### `brew: command not found`

Some macOS installs need Homebrew. Toolkit doesn't auto-install it. Run:

` ` `bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
` ` `

Then re-run integrations install.

### Linux without `brew` / system Node

Some entries require `npm install -g <tool>`. If you use system Node (`/usr/bin/node`), `npm i -g` may need `sudo` â€” toolkit refuses to elevate. Solutions:
- Use `nvm` or `asdf` for user-space Node.
- Or install via vendor's user-space tarball if listed.
- Or skip the CLI; use just the MCP.

### AWS shared CLI

Both `aws-cost-explorer` and `aws-cloudwatch-logs` reference the same `aws` CLI. The installer detects this and runs `aws` install only once per TUI session.

### Post-install hints don't auto-execute

By design. The toolkit prints `â†’ Next: aws configure` etc., but never opens browsers or runs auth flows on your behalf. Run them yourself.

### `--mcps is deprecated`

Phase 32 added `--integrations` as the canonical name. `--mcps` still works (alias) but prints a deprecation note. Switch when convenient; no rush â€” alias removal is post-v5.0.
```
