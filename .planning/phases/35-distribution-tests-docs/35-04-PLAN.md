Above existing `[4.8.0]`, add:

```markdown
## [4.9.0] — 2026-05-02

### Added

- **Integrations Catalog** — unified MCP + CLI install page (`--integrations`) with:
  - Category grouping (10 categories: docs-research, backend, payments, email, workspace, project-management, communication, design, dev-tools, monitoring)
  - Per-component status detection (MCP via `claude mcp list`, CLI via `command -v`)
  - `unofficial: true` confirm gate for community/browser-automation entries (notebooklm, telegram)
  - `--mcp-only` / `--cli-only` mutex flags for component-scoped installs
  - Closing summary table (Entry × MCP × CLI matrix with totals)
- **12 new integrations**:
  - **Backend**: supabase (MCP+CLI), cloudflare (MCP+wrangler), aws-cost-explorer (MCP), aws-cloudwatch-logs (MCP+shared `aws` CLI)
  - **Payments**: stripe (MCP+CLI)
  - **Project Management**: youtrack, linear, jira (MCP only)
  - **Design**: figma (MCP only)
  - **Communication**: slack (MCP), telegram (MCP, unofficial)
  - **Docs/Research**: notebooklm (MCP, unofficial)
- **CLI installer library** `scripts/lib/cli-installer.sh` exposing `cli_detect`, `cli_install`, `cli_post_install_hint` (cross-platform via `uname -s` dispatch, no sudo, brew-absent fallback)
- **Schema validator** `scripts/validate-integrations-catalog.py` (Python stdlib only) wired into `make check` and CI
- **Three hermetic test suites**: `test-integrations-catalog.sh` (≥10 assertions), `test-cli-installer.sh` (≥8), `test-integrations-tui.sh` (≥15 new), wired into Makefile + CI as Tests 32-34
- **`docs/INTEGRATIONS.md`** — complete catalog reference with category groups, install flow, OAuth setup links, troubleshooting, and Global-vs-per-project boundary

### Changed

- **Schema migration**: `scripts/lib/mcp-catalog.json` → `scripts/lib/integrations-catalog.json` (schema_version: 2) with new top-level structure: `categories[]`, `components.mcp{}`, `components.cli{}`. 9 surviving existing entries (context7, firecrawl, magic, notion, openrouter, playwright, resend, sentry) tagged with category. Optional CLI blocks added to firecrawl, playwright, sentry whose CLIs add real value.
- **Backward compat**: `--mcps` flag in `scripts/install.sh` continues to work as alias for `--integrations` with one-line stderr deprecation note (alias removal is post-v5.0).

### Removed

- **`sequential-thinking`** entry from catalog (DROP-01) — native Claude extended thinking covers the use case. Existing user installs unaffected; no migration required.

### Migration notes

- Users on v4.8 → v4.9: re-run `update-claude.sh` to pick up the new lib + script + JSON catalog. v4.4 LIB-01 D-07 jq-path auto-discovery handles all three with zero new code. Any existing `sequential-thinking` MCP in user's `claude mcp list` keeps working — toolkit doesn't auto-uninstall MCPs (boundary preserved).
```
