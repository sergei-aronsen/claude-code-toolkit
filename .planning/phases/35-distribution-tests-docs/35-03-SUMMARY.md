5. **OAuth / auth setup** — per-entry env-var names + auth commands, plus `~/.claude/mcp-config.env` mention.
6. **Global vs per-project (DOCS-02)** — contrasts MCP/CLI/keys (global) vs SDK/project config (per-project), with a Layer × Where × Example table.
7. **Troubleshooting** — `brew: command not found` on macOS, Linux no-brew/system-Node, AWS shared CLI, post-install hints don't auto-execute, `--mcps deprecated`, MCP-registered-without-key recovery flow.
8. **Adding new entries** — required MCP/CLI fields + the validator + test commands needed when bumping the catalog.
9. **See also** — cross-links to INSTALL.md, MCP-SETUP.md, CHANGELOG.md, and the catalog JSON.

### Catalog table source-of-truth

The 20-entry catalog table was generated by reading `scripts/lib/integrations-catalog.json` directly (CONTEXT.md D-19). Key fields in each row:

- **MCP package** — extracted from `install_args[]` (skipping runners `npx` / `-y` / `uvx` / `@latest`)
- **Companion CLI** — `components.cli.<name>.detect_cmd` if present, else `—`
- **Auth** — `OAuth (browser)` if `requires_oauth: true`; else comma-list of `env_var_keys`; else `—`
- **Notes** — `unofficial` flag mirrored verbatim, plus AWS shared-CLI annotation per the troubleshooting section

Real package names (verified against the JSON, not plan placeholders):

- `@stripe/mcp` (not `@stripe/mcp-server`)
- `youtrack-mcp` (not `@youtrack/mcp`)
- `slack-mcp-server` (not `@slack/mcp-server`)
- `figma-developer-mcp` (not `@figma/mcp`)
- `awslabs.cloudwatch-logs-mcp-server@latest` and `awslabs.cost-explorer-mcp-server@latest`

### DOCS-02 boundary statement

Quotes the toolkit's contract verbatim: **"This catalog installs dev-machine globals — stuff Claude Code needs to call mid-conversation."** Then enumerates what stays per-project (`@supabase/supabase-js`, `stripe-node`, `@aws-sdk/client-s3`, `@notionhq/client`) and provides the Layer × Where × Example reference table. Closing rule: *"If you find yourself wanting to install an SDK via this toolkit, the answer is no — that's your project's concern."*
