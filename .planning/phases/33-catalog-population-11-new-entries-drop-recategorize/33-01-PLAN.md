# Plan 33-01: Backend Cluster

## Goal

Add 4 new entries to `scripts/lib/integrations-catalog.json`, all category `backend`:

- **INT-01:** `supabase` (MCP + CLI)
- **INT-02:** `cloudflare` (MCP + CLI `wrangler`)

- **INT-05:** `aws-cloudwatch-logs` (MCP + shared `aws` CLI block)

### Task 1: Verify MCP package names exist on npm/PyPI

Use `WebFetch` (or `npm view <pkg>`) to verify each candidate. For each entry, pick the first candidate that:
- Exists in registry
- Has been updated within last 12 months (sanity check)

Candidates (try in order):

- **supabase**: `@supabase/mcp-server-supabase`, then `@supabase/mcp-server`, then `supabase-mcp`
- **cloudflare**: `@cloudflare/mcp-server-cloudflare`, then any of `cloudflare-mcp`, `@cloudflare/mcp-server-bindings`. AWARE: Cloudflare publishes MULTIPLE MCPs (Workers, R2, KV, D1, Logs). Pick the most general-purpose for the catalog default, document narrower alternatives in entry `description`.
- **aws-cost-explorer**: `awslabs.cost-explorer-mcp-server` (PyPI via `uvx`), or any official npm equivalent. If only PyPI exists, install_args becomes `["aws-cost-explorer", "--", "uvx", "awslabs.cost-explorer-mcp-server@latest"]`.
- **aws-cloudwatch-logs**: `awslabs.cloudwatch-logs-mcp-server` (same uvx pattern).

If a package can't be verified after 3 candidates, leave the entry OUT of this commit and document in SUMMARY.md as "deferred — no verifiable package as of execution date".

### Task 3: Resolve linux install commands without sudo

Per CONTEXT.md D-13: NO `sudo`. For each entry's `install.linux`:

- **supabase linux:** Try `npm install -g supabase` if package exists, else `curl -fsSL https://supabase.com/install.sh | sh` if vendor installer exists, else hint-only (delete `cli` block, just keep MCP + put install instructions in MCP description).
- **cloudflare linux:** `npm install -g wrangler` works if Node uses user-space (nvm/asdf). Document in hint.
- **aws linux:** Vendor's official user-space install: `curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o /tmp/awscliv2.zip && unzip -q /tmp/awscliv2.zip -d /tmp && /tmp/aws/install --install-dir ~/.local/aws-cli --bin-dir ~/.local/bin && rm -rf /tmp/aws /tmp/awscliv2.zip`. Long but no sudo.

If linux install becomes too complex (> 200 chars), fall back to hint-only and document.
