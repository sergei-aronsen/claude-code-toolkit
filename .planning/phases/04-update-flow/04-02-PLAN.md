<objective>
Delete the hand-maintained file lists in `scripts/update-claude.sh` and replace them with a manifest-driven loop that iterates `manifest.files.*` filtered by `compute_skip_set <mode>`. Detect new/removed/modified files via jq set-math over `state.installed_files[]` vs `manifest.files.*`, then dispatch each with the CONTEXT-locked UX: auto-install new, batch-prompt removed, per-file `[y/N/d]` for modified (with `diff -u`).

Purpose: This is the STRUCTURAL fix to BUG-07-class drift (UPDATE-02). The current `update-claude.sh` has 29 command filenames, 7 prompts, 4 agents, 10 skills hand-written into `for` loops — whenever someone adds a file to `manifest.json` but forgets to add it to `update-claude.sh`, existing v3.x users don't get the file on update. Deleting the hand-lists eliminates the drift surface entirely.

**New helper to add — `compute_file_diffs_obj` in `scripts/lib/install.sh`:**

```bash
# compute_file_diffs_obj <state_json> <manifest_path> <mode>
# Emits ONE JSON object to stdout:
#   { "new": [...], "removed": [...], "modified_candidates": [...] }
# Where:
#   new                 = (manifest.files.* - state.installed_files) - skip_set(mode)
#   removed             = state.installed_files - manifest.files.*
#   modified_candidates = state.installed_files ∩ manifest.files.*
# Returns 0 on success, 1 on unknown mode (forwarded from compute_skip_set).
compute_file_diffs_obj() {
    local state_json="$1" manifest_path="$2" mode="$3"
    local mp ip sp
    mp=$(jq -c '[.files | to_entries[] | .value[] | .path]' "$manifest_path") || return 1
    ip=$(jq -c '[.installed_files[].path]' <<<"$state_json")                  || return 1
    sp=$(compute_skip_set "$mode" "$manifest_path")                             || return 1
    jq -nc --argjson m "$mp" --argjson i "$ip" --argjson s "$sp" \
         '{ new: (($m - $i) - $s),
            removed: ($i - $m),
            modified_candidates: [$i[] | select(. as $x | $m | index($x) != null)] }'
}
```

**Research-verified patterns (MUST mirror):**

- Pipeline + `< <(...)` process substitution (never pipe `jq | while read`) — `init-local.sh:253-281`. Counters lost in subshell (Pitfall 1).
- `diff -u local remote || true` — `diff` returns 1 when files differ, which is expected; `set -e` aborts otherwise (Pitfall 2).
- `read -r -p "..." ans < /dev/tty 2>/dev/null; ans="${ans:-N}"` — fail-closed prompt (matches `init-claude.sh:126/151/178/219`, Phase 3 D-42).
- For modified-file diff prompt, download remote once to a tempfile inside the `d` path, reuse for overwrite (avoid double curl). Remote URL is `$REPO_URL/<path>`.

**Skip when state.sha256 is empty:** per RESEARCH Pitfall 11, an empty `sha256` in state means "unknown install-time hash" (Phase 3 write_state edge case) — treat as SKIP silently, do NOT prompt.

            return 0

        done
