## Dependencies between phases

```text
Phase 1 (trim) → Phase 2 (additions) — kill before add to avoid conflicts
Phase 2 → Phase 3 (hooks) — hooks reference components from phase 2
Phase 3 → Phase 4 (templates) — independent, can be parallel BUT sequential safer
Phase 4 → Phase 5 (external) — installer changes assume phase 4 structure
Phase 5 → Phase 6 (docs) — describe final state
Phase 6 → Phase 7 (release) — release requires all PRs merged
```

Strict sequential. No parallel branches in this milestone.

---

## Per-phase exit criteria

Each phase must satisfy before moving to next:
- All tasks completed
- `make check` passes (markdownlint + shellcheck + validate)
- PR merged to main (squash or rebase, project preference)
- Coverage gate: every REQ-ID this phase claims is met has evidence in artifacts
- For HIGH-risk phases: /council approval recorded in artifacts
