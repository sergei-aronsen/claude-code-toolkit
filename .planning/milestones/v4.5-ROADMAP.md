## Key Decisions

- **Mirror as snapshot, not on-demand fetch** (Phase 26) — skills committed to repo for offline-installability.
- **Symlinks for sub-plugin trees** (Phase 27) — zero duplication; `plugins/tk-skills/skills` → `../../templates/skills-marketplace/`.
- **Marketplace.json plugin entries omit version** (Phase 27 MKT-02) — `plugin.json` is single source of truth.
- **Per-MCP wizard with `read -rs`** (Phase 25) — secrets never echoed; `~/.claude/mcp-config.env` mode 0600.
- **`is_*_installed` 3-state for MCPs / 2-state for skills** (Phases 25, 26) — MCP needs CLI-absent state; skills are filesystem-only.
