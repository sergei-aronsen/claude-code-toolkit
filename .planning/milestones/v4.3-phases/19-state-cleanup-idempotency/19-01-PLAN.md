<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test sandbox → host filesystem | The test creates `$SANDBOX` under `/tmp/`. Misuse of `rm -rf` outside `$SANDBOX` would be a host-impact risk. |
| Test stdin → uninstall.sh | The test does not inject stdin (no TTY interaction in no-op path). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01-01 | Tampering | `trap 'rm -rf "$SANDBOX"' EXIT` — `$SANDBOX` could be empty/unset → `rm -rf ""` → host-wide deletion | mitigate | `mktemp -d /tmp/uninstall-idempotency.XXXXXX` always returns a non-empty path on success. The test relies on `set -euo pipefail` to abort if `mktemp` fails before the trap fires. Use `${SANDBOX:?}` in the trap to fail fast if the variable becomes empty: `trap 'rm -f "${MARKER_FILE:?}"; rm -rf "${SANDBOX:?}"' EXIT`. |
| T-19-01-02 | Information disclosure | Test invocation with malicious `TK_UNINSTALL_LIB_DIR` could source attacker-controlled lib files | accept | Test sets `TK_UNINSTALL_LIB_DIR="$REPO_ROOT/scripts/lib"` from a trusted, repo-rooted path. Test runs only in CI / developer machine; not a user-facing attack surface. |
| T-19-01-03 | Denial of service | `mktemp -d /tmp/...` could exhaust /tmp inodes on parallel CI runs | accept | `mktemp -d` cleanup is via EXIT trap; even if 100 parallel runs, /tmp inode pressure is negligible. CI infrastructure problem, not code problem. |

**Severity:** No HIGH findings. All threats mitigated or accepted with rationale.
