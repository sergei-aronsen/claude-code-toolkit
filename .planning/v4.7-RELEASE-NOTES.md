# v4.7.0 — Council Rework (Phase 24 SP2-11)

> Draft release notes for the GitHub Release. Paste verbatim into
> `gh release create v4.7.0 --notes-file .planning/v4.7-RELEASE-NOTES.md`
> (or via the web UI) after PR #13 lands on `main`.

The Supreme Council went from a thin Gemini+ChatGPT plan-validator to
a full review pipeline with editable prompts, secret redaction,
content-hash caching, cost tracking, multi-locale prompts, and an MCP
server for Claude Desktop. Existing `/council` invocations keep
working; everything new is opt-in.

## Highlights

### Editable prompts + persona overlays

System prompts now live as files at `~/.claude/council/prompts/*.md`
and pre-pend domain-specific overlays (security, performance, ux,
migration) when `detect_domain()` matches the plan. Edit the installed
copies directly — the `.upstream-new.md` sidecar pattern preserves
your edits across updates. **Try:**

```bash
COUNCIL_DEBUG=1 brain "fix auth token rotation"
# → persona: role=skeptic domain=security overlay=loaded
```

### Cost tracking + cache

Every Council call appends one JSON row to
`~/.claude/council/usage.jsonl` with provider, model, tokens, and
dollar cost. New `/council-stats` slash command renders human or CSV
summaries. Identical requests within TTL hit the content-hash cache
and cost zero. **Try:**

```bash
brain "<plan>"        # first run — real provider call
brain "<plan>"        # second run — [cached <ts>] marker, zero cost
brain --no-cache "<plan>"  # bypass for one run
/council-stats --month --csv
```

Optional cost-confirm gate via `COUNCIL_COST_CONFIRM_THRESHOLD=0.50`
prompts before any call exceeding the threshold (CI / non-TTY runs
auto-proceed with stderr warning).

### Provider hardening

OpenAI Codex CLI now supported as a $0 alternative to API mode.
Reasoning effort pinned to `high` for the gpt-5.2 / o3 family.
Gemini `thinkingBudget` raised to 32768. New OpenRouter free-model
fallback chain rescues primary backend failures.

### Secret redaction

All context blocks (CLAUDE.md, README, planning docs, recent commits,
TODOs, git diff, files + matching tests) pass through redaction
before transmission. Default patterns cover Stripe live keys,
Anthropic `sk-ant-`, generic high-entropy hex, and `.env` quoted
secrets. Append project-specific patterns to
`~/.claude/council/redaction-patterns.txt`.

### New flags + modes

| Flag | What it does |
|------|--------------|
| `--no-cache` | Bypass the content-hash cache for one run |
| `--dry-run` | Build full prompts, print estimated cost, no API calls |
| `--format json` | Emit single-line JSON for tooling integration |
| `--lang en\|ru\|auto` | Council prompt language; `auto` reads CLAUDE.md |
| `--mode retro --commit <sha>` | Post-implementation alignment check |

### Russian locale

Russian translations of the four system prompts ship under
`~/.claude/council/prompts/ru/`. Auto-selects when CLAUDE.md
Cyrillic ratio exceeds 0.2; force with `--lang ru` / `--lang en`.
Verdict tokens stay English so the parser remains language-agnostic.

### Claude Desktop MCP server

`scripts/council/mcp-server.py` exposes Council as a stdio JSON-RPC
MCP server. `setup-council.sh` offers an interactive step that
registers it in `claude_desktop_config.json`, atomically merging with
any existing servers. After restart, Claude Desktop sees three tools:
`council_validate`, `council_audit_review`, `council_stats`.

### TL;DR auto-summary

Every written `council-report.md` now starts with a 5-second TL;DR
block (verdict + top 3 concerns + detected domain) before the full
reviewer prose.

## Upgrade

```bash
bash <(curl -sSL https://raw.githubusercontent.com/sergei-aronsen/claude-code-toolkit/main/scripts/setup-council.sh)
```

The installer downloads the new prompts, redaction patterns, pricing
table, persona overlays, and Russian translations into
`~/.claude/council/`. Existing local edits are preserved via the
`.upstream-new` sidecar pattern.

For deep documentation including architecture, provider matrix, cost
considerations, and customization guides, see
[`docs/COUNCIL.md`](docs/COUNCIL.md).

## Breaking changes

None. Every new flag and mode is opt-in. Existing `/council "<plan>"`
invocations behave identically (with the addition of a free TL;DR
block in the saved report).
